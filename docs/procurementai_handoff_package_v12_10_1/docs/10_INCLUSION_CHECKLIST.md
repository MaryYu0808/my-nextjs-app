# Inclusion Checklist (Phase 1)

Use this checklist when adding new companies to the directory.

## Step 1 — Procurement relevance gate (must satisfy >= 2)
- [ ] Product positioning (product page, not blog) explicitly mentions: procurement / sourcing / S2P / P2P / CLM / supplier risk / AP automation
- [ ] Product maps clearly into one of the 8 workflow categories (single primary category)
- [ ] Customer stories target procurement roles (CPO / Procurement Ops / Category Manager / AP)
- [ ] Core modules are procurement primitives (RFx, CLM, supplier onboarding/risk, invoice/AP, intake/orchestration, negotiation support, spend analytics)

If fewer than 2 are satisfied: **exclude (Tier 3 / Secondary Layer)**.

## Step 2 — Tier decision
- Tier 1: procurement-native platform or procurement-focused AI specialist
- Tier 2: procurement-first adjacent intelligence (risk/ESG/due diligence/cost intelligence) with procurement as a primary buyer/use case

## Step 3 — Data completeness (minimum)
- [ ] Name
- [ ] Website
- [ ] HQ country
- [ ] Primary workflow category (single)
- [ ] Use cases (1–5)
- [ ] Automation maturity (one of 4)
- [ ] Short description
- [ ] At least 1 public source URL
- [ ] Data last verified timestamp

## Step 4 — Quality notes
- Keep language vendor-neutral
- Do not imply ranking or endorsement
- Prefer official product pages for sources
